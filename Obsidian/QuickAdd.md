QuickAdd 插件，本质上是一个 **“动作引擎”** ，把一系列需要手动操作的步骤，变成一个可以一键执行的自动化命令。

### 📦 安装与设置
1.  在 Obsidian 的「设置」→「第三方插件」→「社区插件」中搜索 “**QuickAdd**” 并安装、启用。
2.  启用后，你会在 Obsidian 设置页面的最下方看到 **QuickAdd** 的配置项。
3.  在命令面板（`Cmd/Ctrl+P`）中搜索 “QuickAdd: Run QuickAdd” 即可运行创建好的命令。

### 🔧 核心功能
QuickAdd 的核心在于四种“**Choice**”（选择项），它们是你实现不同自动化的基石。你可以把它们理解为四种不同功能的“场景预设”，下表清晰地展示了它们的区别：

| 功能 | 核心作用 | 典型场景 |
| :--- | :--- | :--- |
| **⚡️ Template (模板)** | 基于模板标准化创建新笔记。 | 会议纪要、读书笔记、项目周报 |
| **📥 Capture (捕获)** | 不打断思路，快速记录并追加到现有笔记中。 | 闪念笔记、任务清单、日记补充 |
| **⚙️ Macro (宏)** | 将多个动作打包成一步执行的自动化流程。 | 创建文件夹→建立子笔记→生成索引 |
| **🗂️ Multi (多选)** | 把多个 Choice 打包进一个可选菜单，进行“二次分发”。 | 拥有多个模板时呼出一个命令菜单选择 |

#### ⚡️ Template (模板)：一键生成
Template 的核心是将“新建笔记”的动作标准化。使用时，首先需要提前备好一个 `.md` 模板文件。

*   **核心配置**：设置 `Template Path`（模板路径）；设置 `File Name Format`（文件名格式），如 `{{DATE:YYYY-MM-DD}}_笔记标题`；设置 `Create in folder`（创建位置）。
*   **模板变量**：使用 `{{DATE:YYYY-MM-DD}}` 插入动态日期，或使用 `{{VALUE:提示语}}` 在运行时弹窗让用户输入书名等信息。变量可以组合使用，比如 `{{VALUE:书名}}_{{DATE:YYYY}}`。
*   **进阶用法**：
    *   **父子结构**：在 `File Name Format` 中填入 `主文件夹/{{VALUE:子文件夹名}}/{{VALUE:文件名}}`，即可一次性创建文件夹结构和笔记。
    *   **父子笔记**：在模板文件内容中预留 `[[{{VALUE:子笔记名称}}]]`，可让当前笔记自动链接到即将创建的子笔记。
    *   **多种文件类型**：模板不仅支持 `.md` 文件，也支持 `.canvas`（画布）和 `.base`（数据库）等格式。

#### 📥 Capture (捕获)：快速记录
相比于需要创建新文件的 Template，Capture 的核心是“追加内容”到某个现有文件，全程不打断当前工作流。

*   **核心配置**：设置 `Capture To`（目标文件），如 `日记/每日日志.md`，内容会写入这个文件。设置 `Capture Format`（输入格式），输入格式可以混合纯文本和变量，例如 `- [ ] {{VALUE:待办事项}} 📅 {{DATE:YYYY-MM-DD}}`。
*   **变量运用**：
    *   `{{VALUE}}`：运行时会弹出对话框让你输入。
    *   `{{SELECTED}}`：自动获取你在当前笔记中选中的文字，无需重复输入。
*   **配置项详解**：
    *   **Position (写入位置)**：`Top`（文件顶部）、`Bottom`（文件底部）、`After Heading`（指定标题下）等。
    *   **Append Link**：设置是否自动创建链接。
    *   **Create file if it doesn't exist**：如果目标文件不存在则自动创建。
*   **智能目标**：`Capture To` 甚至支持填入**文件夹路径**或**标签名称**（如 `#待办`），运行时会让你从该文件夹/标签下选择要写入的文件。

#### ⚙️ Macro (宏)：组合动作
Macro 是 QuickAdd 最强大的功能，它将多个步骤打包成一个工作流。Macro 和 Macro Choice 是母鸡与鸡蛋的关系：**Macro Choice** 是最终供你点击运行的“命令”，而 **Macro** 是这个命令背后需要执行的“具体动作步骤”。

*   **创建流程**：在 QuickAdd 设置中点击 **“Manage Macros”**，创建一个 **Macro**（宏），并添加需要依次执行的**Commands**（命令）。然后回到主界面，创建一个 **Macro Choice**，在设置中指向刚才配置好的 **Macro**。
*   **命令类型**：一个宏中可以混用多种命令：
    *   **Obsidian Command**：执行 Obsidian 内部命令。
    *   **User Script**：运行自定义 JavaScript 脚本，能访问 Obsidian API，是进阶自动化的核心。
    *   **Nested Choice**：嵌套另一个 QuickAdd Choice，实现模块化复用。
    *   **AI Assistant**、**Conditional**等更多丰富选项。

#### 🗂️ Multi (多选)：管理菜单
Multi 是一个“命令收纳盒”，用来把多个 Template、Capture、Macro Choice 聚合在一起，形成菜单。

*   **配置方法**：创建一个 **Multi Choice**，在列表中，将其他 Choice **鼠标拖拽到它下方并向右侧缩进**，即可将其收纳。
*   **适用场景**：拥有多种笔记模板时，用 Multi 打包成一个“新建笔记”菜单；或者整合多个不同的 Capture 操作等。通常会为 Multi Choice 分配唯一的全局快捷键，快速呼出命令菜单。

### 💎 实用技巧
*   **理解图标与命名**：一个成熟的 QuickAdd 命令通常包含一个 **⚡️闪电图标**（出现在命令面板），**一个由⚙️齿轮图标配置的 Choice**，以及一个你输入的、能在命令面板或菜单中清晰理解的**名称**。官方文档建议将 Choice 的类型（Template/Capture）包含在名称中，方便管理。
*   **快捷键绑定**：创建 Choice 后，点亮其⚡️图标，即可在 Obsidian 的「快捷键」设置中为其绑定全局热键，例如 `Alt + M` 一键开始会议记录。
*   **代码调试**：可以在编写的 JavaScript 脚本中使用 `console.log()` 输出信息，然后在 Obsidian 中按 `Ctrl+Shift+I` 打开开发者工具的 Console 面板查看。
